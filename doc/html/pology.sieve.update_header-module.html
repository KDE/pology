Module update_header

Initialize and update the header of a PO file.

When work on a pristine PO starts, or an existing PO is to be revised with new translations, this sieve can be used to automatically set PO header fields to proper values. The revision date is taken as current, while the rest of the info is pulled from user's configuration (~/.pologyrc).

Sieve options:

  • proj:<project_id>: ID of the project
  • nosync: do not issue sync request

Parameter proj specifies the ID of the project which covers the POs about to be operated on. This ID is used as the name of configuration section [project-<ID>], which contains the project data fields. Also used are the fields under the [user] section, unless overriden in the project's section. All the used config fields are:

  • [project-*]/name or [user]/name: user's name
  • [project-*]/email or [user]/email: user's email address
  • [project-*]/language: language code (IS0 639)
  • [project-*]/language-team: human-readable language name
  • [project-*]/team-email: team's email
  • [project-*]/encoding: encoding of PO files
  • [project-*]/plural-forms: the PO plural header (nplurals=...; ...;)

Non-default header fields are not touched, except the revision date and the last translator which are always updated (including the comment line, where translators are listed with years of contributions).

Option nosync is there for cases when this sieve is used in a chain with other sieves, to allow modifying the header only if a catalog got modifed by another sieve that did request syncing.
