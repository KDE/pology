Module check_xml_docbook4

Lightweight validity checking of POs containing Docbook XML.

Docbook is a complex XML application, and nothing short of full validation of XML files generated from translated POs can show if markup is valid. This sieve just checks for well-formedness and mere existance of a tag in Docbook subset, and on the level of single PO message. But this is already enough to catch most of the typical translation errors.

Sieve options:

  • showmsg: show content of the message, with errors highlighted
  • lokalize: open catalogs at failed messages in Lokalize

A message can be excluded from checking by adding no-check-markup sieve flag.
