Module tag_incomplete

Mark fuzzy and untranslated messages as incomplete.

Some people like to translate with non-dedicated tools, such as an ordinary text editor, which may provide no more support for editing PO files other than e.g. syntax highlighting. In such a scenario, this sieve can be used to equip both fuzzy and untranslated messages with incomplete flag, for easy lookup in the editor.

Note that incomplete flags, being custom, will be lost when the PO file is merged with the template next time. This is intentional: the only purpose of the flag is for immediate editing, and the translator may forget to remove some. There is no reason for the flags to persist in that case. Also, if an incomplete flag is forgotten when the message is translated, the subsequent run of this sieve will remove the flag.

Sieve options:

  • branch:<branch_id>: consider only messages from this branch (summit)
  • strip: instead of adding, strip any incomplete flags

For summited catalogs, the branch option is used to restrict modifications to messages from the given branch only. Several branch IDs may be given as a comma-separated list.
