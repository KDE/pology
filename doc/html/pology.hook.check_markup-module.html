Function Details

check_xml(strict=False, entities={}, entpathenv=None, fcap=True, mkeyw=None)

 

Check general XML markup in translations [hook factory].

Text is only checked to be well-formed XML, and possibly also whether encountered entities are defined. Markup errors are reported to stdout.

msgstr can be either checked only if the msgid is valid itself, or regardless of the validity of the original. This is governed by the strict parameter.

A (name, value) dictionary of entities, in addition to XML's default (&lt;, etc.), may be provided using the entities parameter. If it is given as None, entities are ignored by the check.

Entities may also be automatically collected, by parsing all *.entities files in directory paths given by the environment variable entpathenv; entity files are searched for recursively down each directory path. See parse_entities for the format of *.entities files.

If an entity with the first letter in uppercase is encountered and not among the defined ones, it may be allowed to pass the check by setting the fcap parameter to True.

If a message has pipe flag no-check-xml, the check is skipped for that message. If one or several markup keywords are given as mkeyw parameter, check is skipped for all messages in a catalog which does not report one of the given keywords by its markup() method. See set_markup() for list of markup keywords recognized at the moment.

Parameters:
  • strict (bool) - whether to require valid msgstr even if msgid is not
  • entities (dict or None) - additional entities to consider as known
  • entpathenv (string) - environment variable with paths to entity definitions
  • fcap (bool) - whether to allow first-uppercase entities
  • mkeyw (string or list of strings) - markup keywords for taking catalogs into account
Returns: (msgstr, msg, cat) -> numerr
