Module bad_patterns

Check for presence of bad patterns in translation.

Sometimes there are simply definable patterns that should never appear in translation, such as common grammar or orthographical errors. This sieve allows checking for such patterns, either through substring matching or regular expressions. Patterns can be given as parameters, or, more conveniently, read from files.

Sieve parameters:

  • pattern:<string>: pattern to check against
  • fromfile:<path>: file from which to read patterns
  • rxmatch: patterns should be treated as regular expressions
  • casesens: patterns should be treated as case-sensitive

Any number of pattern and fromfile parameters may be given. By default, patterns are matched as substrings, and rxmatch parameter can be issued to consider patterns as regular expressions.

Note: This sieve is deprecated; instead use the check-rules, which provides much more options for defining, matching, and reporting problems.
