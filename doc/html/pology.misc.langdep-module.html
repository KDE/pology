get_module(lang, path, abort=False)

 

Import a language-dependent module.

Module is specified by the language code, and the elements of the relative path in Pology's package for that language. Language can also be None, in which case a language-independent module is looked for. For example:

   get_module("sr", ["hook", "wconv"])

will try to import the pology.l10n.sr.wconv module, while:

   get_module(None, ["hook", "remove-subs"])

will try to import the pology.hook.remove_subs.

The elements of the module path can also contain hyphens, which will be converted into underscores when looking for the module.

If the module cannot be imported, if abort is True the execution will abort with an error message; otherwise an exception is raised.

Parameters:
  • lang (string or None) - language code
  • path (list of strings) - relative module location
  • abort (bool) - whether to abort execution if the module cannot be imported
Returns: module or None
imported module

split_req(langreq, abort=False)

 

Split string of the language-dependent item request.

The language-dependent item request is string of the form [lang:]path[/item][~args], which is to be parsed into (lang, path, item, args) tuple. If language is not stated, its value in the tuple will be None, and likewise for the item and argument strings. The language should be a proper language code, the path a sequence of identifier-like strings connected by dots, item also an identifier-like string, and arguments can be an arbitrary string.

If the item request cannot be parsed, either the execution is aborted with an error message, or an exception is raised, depending on value of abort.

Parameters:
  • langreq (string) - request specification
  • abort (bool) - whether to abort execution or if the request cannot be parsed
Returns: (string or None, string, string or None, string or None)
parsed language and request

get_hook(lang, hmod, func=None, args=None, abort=False)

 

Fetch a language-dependent hook function.

Loads the hook function from pology.l10n.<lang>.hook.<hmod> module. If func is None, the function name defaults to module name; if func is not None, but function of that name is not found, then the function named <hmod>_<func> is additionally tried. If args is not None, then the loaded function is considered a hook factory, and the hook is created by calling it with args string as argument list (it should have no surrounding parenthesis).

Parameters:
  • lang (string) - language code
  • hmod (string) - hook module
  • func (string) - hook of hook factory function name
  • args (string) - argument string to hook factory
  • abort (bool) - if the hook is not loadable, abort or report None
Returns:
the hook

get_hook_lreq(langreq, abort=False)

 

Like get_hook, but the hook is specified as language request.

For a module pology.hook.FOO which defines the FOO() hook function, the hook specification is simply FOO. If the hook function is named BAR() instead of FOO(), the hook specification is given as FOO/BAR; if the hook function is named FOO_BAR(), i.e. the specification would be FOO/FOO_BAR, it can be folded to FOO/BAR. Language-specific hooks (pology.l10n.LANG.hook.FOO) are aditionally preceded by the language code with colon, as LANG:FOO or LANG:FOO/BAR.

If the hook is not a plain hook, but a hook factory function, the factory arguments are supplied after the basic hook specification, separated by tilde: LANG:FOO/BAR~ARGLIST (where LANG: and /BAR may be omitted under previous conditions). Argument list is formatted just like it would be passed in Python code to the factory function, omitting the surrounding parenthesis.

get_result(lang, mod, func=None, args='', abort=False)

 

Fetch a result of language-dependent function evaluation.

Executes function loaded from a pology.l10n.<lang>.<mod> module and returns its result. If func is not given, the function name defaults to run. args is the string representing the argument list to the function call (without surrounding parenthesis).

Parameters:
  • lang (string) - language code
  • mod (string) - language-dependent module
  • func (string) - function name within the module
  • args (string) - argument string to function call
  • abort (bool) - if the function is not found, abort or report None
Returns:
the value returned by the function call
