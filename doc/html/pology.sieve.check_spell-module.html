Module check_spell

Spell-check messages using GNU Aspell (http://aspell.net/).

For spell-checking, messages are split into words, which are then fed to Aspell one by one. Misspelled words can be reported to stdout (either verbosely with suggestions, or succinctly as pure list), or into a rich XML file with a lot of context information.

Sieve options:

  • lang:<language>: language of the Aspell dictionary
  • enc:<encoding>: encoding for text sent to Aspell
  • var:<variety>: variety of the Aspell dictionary
  • list: only report wrong words to stdout, one per line
  • xml:<filename>: build XML report file
  • accel:<char>: strip this character as accelerator marker
  • skip:<regex>: do not check words which match given regular expression
  • skipctxt:<regex>: do not check messages with context that matches given regular expression
  • filter:[<lang>:]<name>,...: apply filters prior to spell checking
  • env:<environment>: environment of internal dictionary supplements
  • suponly: do not use default dictionary, only internal supplements

When dictionary language, encoding, or variety are not explicitly given, they are extracted, in the following order of priority, from: current PO file (language only), user configuration, current system locale (language and encoding only).

If accelerator character is not explicitly given, it may be inferred from the PO header; otherwise, some usual accelerator characters are removed by default.

The filter option specifies text-transformation filters to apply before the text is spell-checked. These are the filters found in pology.filters and pology.l10n.<lang>.filters, and are specified as comma-separated list of [<lang>:]<name> (language stated when a filter is language-specific).

Pology internally collects language-specific supplementary word lists to default Aspell dictionaries, under l10n/<lang>/spell/ directory. These contain either the words that should enter the default dictionary but have not been added yet, or, more importantly, the words that are specific to a given translation environment, i.e. too specific to enter the default dictionary. The env option is used to specify the environment for which the supplementary word lists are loaded, as a subpath l10n/<lang>/spell/<env>: all word lists in that subpath and above will be loaded. This means that the word lists are hierarchical, so that all-environment lists (loaded even when env option is not given) reside directly in l10n/<lang>/spell/, and the more specific ones in subdirectories below it. The defaul dictionary can be avoided alltogether, and only supplementary used instead, by giving the suponly option.

The following user configuration fields are considered:

  • [aspell]/language: language of the Aspell dictionary
  • [aspell]/encoding: encoding for text sent to Aspell
  • [aspell]/variety: variety of the Aspell dictionary
  • [aspell]/supplements-only: use only internal dictionaries
