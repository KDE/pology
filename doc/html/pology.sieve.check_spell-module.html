Module check_spell

Spell-check messages using GNU Aspell (http://aspell.net/).

For spell-checking, messages are split into words, which are then fed to Aspell one by one. Misspelled words can be reported to stdout (either verbosely with suggestions, or succinctly as pure list), or into a rich XML file with a lot of context information.

Sieve options:

  • lang:<language>: language of the translation
  • enc:<encoding>: encoding for text sent to Aspell
  • variety:<varname>: variety of the language dictionary
  • list: only report wrong words to stdout, one per line
  • xml:<filename>: build XML report file
  • accel:<char>: strip this character as accelerator marker
  • skip:<regex>: do not check words which match given regular expression

When language or encoding are not explicitly given, they are extracted from the current system locale.

If accelerator character is not explicitly given, it may be inferred from the PO header; otherwise, some usual accelerator characters are removed by default.
