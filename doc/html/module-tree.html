Module Hierarchy

  • pology: Pology is a framework for custom processing of PO files in field environments, and a collection of tools based on its foundation, including many smaller scripts and subscripts which can be used to perform various precision tasks.
    • pology.file: Interfaces to PO files.
      • pology.file.catalog: Collection of PO entries.
      • pology.file.header: Header entry in PO catalogs.
      • pology.file.message: Message entries in PO catalogs.
    • pology.hook
      • pology.hook.bad_patterns: Detect unwanted patterns in translation.
      • pology.hook.check_xml: Check validity of XML markup.
      • pology.hook.gettext_tools: Pipe PO files through Gettext commands.
      • pology.hook.remove_subs: Remove special substrings from parts of the message.
    • pology.l10n: Language-specific Pology tools, arranged in submodules by language code.
      • pology.l10n.fr: Language-specific tools for French language.
        • pology.l10n.fr.hook: Hooks specific for French.
          • pology.l10n.fr.hook.patternAccents: Accent equivalence in regular expression patterns.
          • pology.l10n.fr.hook.patternEntities: Convert special entities in rule patterns.
        • pology.l10n.fr.sieve: Sieves specific to French language.
          • pology.l10n.fr.sieve.setUbsp
      • pology.l10n.ja: Language-specific tools for Japanese language.
        • pology.l10n.ja.hook: Hooks specific for Japanese.
          • pology.l10n.ja.hook.katakana: Retain only Katakana words in the text, separated by spaces.
      • pology.l10n.sr: Language-specific tools for Serbian language.
        • pology.l10n.sr.hook: Hooks specific for Serbian.
          • pology.l10n.sr.hook.checks: Various checks for translations into Serbian.
          • pology.l10n.sr.hook.cyr2lat: Transliterate Cyrillic text into Latin.
          • pology.l10n.sr.hook.ijeref: Process iyekavian text with marked yat-reflexes.
    • pology.misc: Various functionality for processing contents of PO files.
      • pology.misc.colors: Standard codes for shell colors.
      • pology.misc.comments: Parsing and composing message comments.
      • pology.misc.config: Access user configuration for Pology.
      • pology.misc.diff: Producing diffs between texts.
      • pology.misc.escape: Escaping texts in various contexts.
      • pology.misc.fsops: Operations with file system and external commands.
      • pology.misc.langdep: Fetch language-dependent modules, functions, data, etc.
      • pology.misc.markup: Process text markup.
      • pology.misc.monitored: Framework for monitored classes.
      • pology.misc.report: Report info, warning and error messages.
      • pology.misc.resolve: Replacement of segments in text which define some underlying values.
      • pology.misc.rules: Match messages by rules of arbitrary specificity.
      • pology.misc.split: Splitting message fields into syntactical elements.
      • pology.misc.tabulate: Pretty-printing of tabular data.
      • pology.misc.timeout: A timeout decorator.
      • pology.misc.vcs: Version control operations.
      • pology.misc.wrap: Text wrapping, with special handling for typical texts in PO files.
    • pology.scripts: Standalone scripts for processing PO files and Pology-related service.
      • pology.scripts.create_rules_from_koffice_autocorrect: Create Pology rules from the KOffice KWord autocorrect xml file.
      • pology.scripts.fallback_import_paths: Insert fallback import paths in scripts.
      • pology.scripts.organizeDict: Organize dictionary file:
      • pology.scripts.poascribe
      • pology.scripts.posieve: Sieve messages in collections of PO files.
      • pology.scripts.posummit
      • pology.scripts.potagsplit
    • pology.sieve: Catalog sieves.
      • pology.sieve.apply_filter: Apply filters to translation.
      • pology.sieve.bad_patterns
      • pology.sieve.check_grammar: Sieves messages with the LanguageTool grammar checker (http://www.languagetool.org)
      • pology.sieve.check_rules: Try to fail messages by rules and warn when that happens.
      • pology.sieve.check_spell: Spell-check messages using GNU Aspell (http://aspell.net/).
      • pology.sieve.check_xml_docbook4: Lightweight validity checking of POs containing Docbook XML.
      • pology.sieve.check_xml_kde4
      • pology.sieve.count_ctxmark
      • pology.sieve.diff_previous
      • pology.sieve.empty_fuzzies: Make all fuzzy entries untranslated.
      • pology.sieve.fancy_quote: Transform ASCII single and double quotes into fancy counterparts.
      • pology.sieve.find_messages: Find messages by regular expression matching.
      • pology.sieve.generate_xml: Generate a XML output from the input PO files.
      • pology.sieve.normctx_ooo: Convert OpenOffice embedded context to Gettext context.
      • pology.sieve.remove_fuzzy_comments: Remove selected manual comments in fuzzy messages.
      • pology.sieve.remove_obsolete
      • pology.sieve.resolve_alternatives
      • pology.sieve.resolve_entities
      • pology.sieve.set_header: Set elements of the PO header.
      • pology.sieve.stats: Statistics: message and word counts by category, etc.
      • pology.sieve.tag_incomplete: Mark fuzzy and untranslated messages as incomplete.
      • pology.sieve.tag_untranslated
      • pology.sieve.unfuzzy_context_only: Unfuzzy those messages fuzzied only due to a context change.
      • pology.sieve.unfuzzy_ctxmark_only: Unfuzzy those messages fuzzied only due to a change in context marker.
      • pology.sieve.unfuzzy_inplace_only
      • pology.sieve.update_header: Initialize and update the header of a PO file.
